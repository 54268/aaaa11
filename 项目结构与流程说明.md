# 项目结构与流程说明

## 1. 这套工程在做什么

这套工程服务于你的开放集 SEI 方案验证，目标是把“已知类学习 + 边界挖掘 + 伪未知生成 + 拒识校准 + 开放集评估”串成一条完整可运行链路。

它遵循你的核心要求：

- 不能只训练关键边界样本
- 必须保留普通边缘样本
- 普通边缘样本按本类外法向外推生成伪未知
- 关键边界样本按竞争边界思路处理
- 最后要有稳定的开放集拒识与性能指标输出

## 2. 与你的方案是否一致

从实现逻辑上，这套代码与方案的对应关系如下：

### 2.1 已知类表征学习

- 使用 CVCNN 提取 I/Q 信号嵌入特征
- 使用原型分类头做已知类判别
- 使用三段特征正则化损失训练：
  - 基础损失
  - 角度损失
  - 原型损失

### 2.2 原型竞争与边界感知

- 每个样本不仅看本类原型距离
- 还看最近异类原型距离
- 边界挖掘时同时考虑局部边缘性和原型竞争间隙

### 2.3 普通边缘样本保留

- 不会丢掉普通边缘样本
- 会按 `ordinary_edge_ratio` 单独保留一批普通边缘样本
- 这些样本沿“样本到本类原型”的外法向生成伪未知

### 2.4 关键边界样本单独处理

- 关键边界样本按得分排序选出
- 生成伪未知时不是只按法向推
- 而是“本类外法向 + 最近异类原型排斥方向”联合生成
- 同时带冲突保护和随机扰动

### 2.5 开放集拒识

- 用 OpenMax 做尾部建模与拒识
- 再和原型距离未知分数做融合
- 通过验证集搜索融合权重和拒识阈值

结论：

- 从处理链路看，这套实现符合你的方案主线
- 且没有把第三方论文代码原样照搬成“你的创新模块”
- 真正属于你方案关键创新的位置，仍然由本地代码自己实现

## 20. 整个模型是怎么训练的

下面按真实执行顺序说明。

### 第 1 步：数据预处理

WiSig 路线：

- 读取官方 `ManySig.pkl`
- 解出其中的嵌套结构
- 转成统一 `npz` 数据格式

Oracle 路线：

- 读取 `sigmf-meta + sigmf-data`
- 自动修正真实复数类型
- 按窗口切成固定长度样本
- 划分已知类训练/验证/测试和未知类测试

### 第 2 步：已知类闭集训练

训练对象：

- CVCNN backbone
- prototype head

训练操作：

- 从 I/Q 信号提取 embedding
- 计算 embedding 到每个类原型的平方欧氏距离
- 用负距离作为分类 logits
- 优化以下损失：
  - 基础损失
  - 角度损失
  - 原型损失
  - 可选 episodic 原型正则
- 每个 batch 后更新原型 EMA
- 每个 epoch 后重新刷新全局原型

### 第 3 步：边界挖掘

对每个已知样本，计算：

- 本类局部边缘性
- 最近异类原型竞争间隙

然后：

- 归一化后融合成边界得分
- 每类取得分最高的样本作为关键边界样本
- 每类再保留一批普通边缘样本

### 第 4 步：伪未知生成

普通边缘样本：

- 沿样本指向本类原型的反方向外推

关键边界样本：

- 取本类外法向
- 取最近异类原型排斥方向
- 做加权融合
- 若两个方向冲突，先投影修正再归一化
- 最后乘局部尺度并加少量随机扰动

### 第 5 步：OpenMax 拟合

- 用已知类正确样本的激活统计尾部分布
- 用 Weibull 做尾部建模
- 得到 OpenMax 未知概率

### 第 6 步：原型距离未知分数计算

- 统计已知类预测正确样本的最小距离分布
- 将最小距离标准化
- 映射成原型距离未知分数

### 第 7 步：融合校准

计算：

- `q_om`：OpenMax 未知分数
- `q_pd`：原型距离未知分数

再做：

- `q_u = λ * q_om + (1 - λ) * q_pd`
- 在验证集上搜索最佳 `λ` 和阈值

### 第 8 步：开放集评估

最终在测试集上输出：

- 已知类正确识别能力
- 未知类拒识能力
- 综合开放集性能

## 22. 指标中文名与含义

| 指标键 | 中文名 | 含义 |
| --- | --- | --- |
| `overall_accuracy` | 总体准确率 | 已知类和未知类一起统计的整体分类正确率 |
| `macro_precision` | 宏平均精确率 | 各类别精确率先分别算，再平均 |
| `macro_recall` | 宏平均召回率 | 各类别召回率先分别算，再平均 |
| `macro_f1` | 宏平均 F1 分数 | 各类别 F1 先分别算，再平均 |
| `weighted_f1` | 加权 F1 分数 | 按类别样本数加权的 F1 |
| `known_accuracy` | 已知类准确率 | 只看已知类样本时的分类准确率 |
| `unknown_precision` | 未知类精确率 | 被判成未知的样本里，真正未知所占比例 |
| `unknown_recall` | 未知类召回率 | 真正未知样本中，被成功拒识为未知的比例 |
| `known_fpr_as_unknown` | 已知类误拒率 | 已知类样本被误判成未知的比例 |
| `unknown_false_accept_rate` | 未知类误接收率 | 未知类样本被错误接收为已知类的比例 |
| `auroc` | 受试者工作特征曲线下面积 | 衡量已知/未知分数区分能力，越大越好 |
| `fpr95` | 95% 召回点假阳率 | 在高未知召回条件下已知类误拒比例，越低越好 |
| `oscr` | 开放集分类率 | 同时考虑已知类正确分类和未知类拒识的综合指标 |
