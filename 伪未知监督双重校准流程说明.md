# 伪未知监督双重校准流程说明

## 1. 为什么新增这个步骤

本文的核心创新之一是：先在已知类特征空间中筛选分类边界样本，再从边界向类外侧外推，生成特征层伪未知样本。

旧流程虽然生成了伪未知，但伪未知主要用于融合权重和拒识阈值的校准，没有直接参与一个可学习模型的训练。这样会削弱“边界建模与伪未知生成”在最终拒识结果中的作用。

当前新增的伪未知监督双重校准器，将伪未知作为“未知”正样本，将已知验证样本作为“已知”负样本，直接训练已知—未知二分类边界。因此，伪未知不再只是辅助选择阈值，而是直接决定最终未知分数。

## 2. 它位于总框架的哪个位置

该步骤位于框架图中的“开放集识别/双重校准”部分，具体位置是：

```text
已知类闭集训练
        ↓
128维嵌入特征与类别原型
        ↓
边界样本筛选
        ↓
特征层外推生成伪未知 ─────────────────────┐
                                          │ 监督训练
测试样本 → 编码器 → OpenMax未知分数 ───────┤
                       原型距离未知分数 ─────┤
                                          ↓
                              伪未知监督双重校准器
                                          ↓
                                   最终未知概率
                                          ↓
                                  按类别阈值判决
                                    ↙           ↘
                                  已知          未知
                                                ↓
                                          未知类细分
```

框架图原有的编码器、原型分类、边界样本筛选、伪未知外推、OpenMax、原型距离和未知类细分均保留。只需在“双重校准”模块内部增加“可学习校准器”，并从“伪未知集”画一条训练监督箭头指向该校准器。

真实未知类不需要画到校准器训练支路。真实未知只位于最右侧或最下方的“最终测试集”输入，用于冻结参数后的最终评价。

## 3. 输入和输出

校准器不直接接收128维嵌入，而接收三个可解释分数：

1. OpenMax未知分数 \(q_{\mathrm{OM},i}\)；
2. 原型距离未知分数 \(q_{\mathrm{PD},i}\)；
3. 两者的线性融合分数 \(q_i^{\mathrm{fusion}}\)。

原有融合公式继续保留：

\[
q_i^{\mathrm{fusion}}
=\lambda q_{\mathrm{OM},i}
+(1-\lambda)q_{\mathrm{PD},i}.
\]

随后使用小型可学习校准器：

\[
\hat q_i
=g_{\theta}\left(
q_{\mathrm{OM},i},
q_{\mathrm{PD},i},
q_i^{\mathrm{fusion}}
\right),
\]

其中：

- \(g_\theta\) 为 `3→8→1` 的小型多层感知机；
- 隐藏层使用 ReLU；
- 输出层使用 Sigmoid；
- \(\hat q_i\in[0,1]\) 是最终未知概率。

因此，原线性融合公式并没有被删除，而是从“最终判决分数”变为可学习校准器的一个输入。

## 4. 训练样本如何构造

### 4.1 已知负样本

使用已知验证集样本：

- 经过冻结编码器得到128维嵌入；
- 计算 OpenMax分数、原型距离分数和线性融合分数；
- 二分类标签设置为0，表示“已知”。

使用验证集而不是闭集训练批次，是为了避免校准器只记住编码器训练阶段过于容易的已知样本。

### 4.2 伪未知正样本

使用边界外推得到的特征层伪未知：

- 从普通边界样本和原型竞争边界样本向类外侧外推；
- 当前五折自动搜索选择的外推尺度为1.5；
- 计算伪未知的三个校准器输入分数；
- 二分类标签设置为1，表示“未知”。

伪未知按照来源类别和伪未知类型均衡采样，避免某一已知类或某一种边界样本支配校准器。

### 4.3 训练目标

已知样本和伪未知样本数量保持平衡，使用二元交叉熵训练：

\[
\mathcal L_{\mathrm{cal}}
=-\frac{1}{N}
\sum_i
\left[
t_i\log\hat q_i
+(1-t_i)\log(1-\hat q_i)
\right],
\]

其中 \(t_i=0\) 表示已知，\(t_i=1\) 表示伪未知。

## 5. 五折留类自动搜索

为了避免使用六个真实未知类调参，Oracle的十个已知类被划分为五折。每折执行：

1. 留出两个已知类，将其临时视为模拟未知；
2. 使用其余八类训练临时编码器、类别原型和OpenMax；
3. 从八类训练嵌入中筛选边界并生成伪未知；
4. 使用八类验证已知和伪未知训练校准器；
5. 两个留出类只负责评价迁移拒识能力，不参与校准器训练。

自动搜索内容包括：

- 伪未知外推尺度；
- 融合权重 \(\lambda\)；
- 校准器训练轮数；
- 随机种子；
- 按类别拒识阈值分位点。

当前自动选中的组合为：

| 参数 | 自动选择结果 |
| --- | ---: |
| 伪未知外推尺度 | 1.5 |
| 融合权重 \(\lambda\) | 1.0 |
| 校准器训练轮数 | 250 |
| 随机种子 | 43 |
| 阈值分位点 | 0.96 |

\(\lambda=1.0\) 表示当前数据上初步融合分数等于OpenMax分数，但原型距离分数仍作为校准器的独立输入，因此原型距离分支没有被删除。

## 6. 正式训练与最终测试

五折确定超参数后：

1. 冻结正式十类编码器、类别原型和OpenMax；
2. 使用正式已知验证集作为负样本；
3. 使用正式边界外推伪未知作为正样本；
4. 按五折选出的参数训练正式校准器；
5. 在已知验证集上将0.96分位点还原为十个按类别阈值；
6. 冻结校准器和阈值；
7. 最后一次读取十类已知测试集和六类真实未知测试集。

六类真实未知没有参与：

- 编码器训练；
- 原型计算；
- OpenMax拟合；
- 校准器训练；
- 外推尺度搜索；
- 融合权重搜索；
- 阈值搜索。

真实未知只用于最终报告测试指标。

## 7. 最终拒识判决

设原型分类器给出的已知预测类别为 \(\hat c_i\)，该类别对应阈值为 \(\tau_{\hat c_i}\)，则：

\[
\hat y_i=
\begin{cases}
\mathrm{unknown}, & \hat q_i>\tau_{\hat c_i},\\
\hat c_i, & \hat q_i\le\tau_{\hat c_i}.
\end{cases}
\]

被判为已知的样本输出对应已知发射源类别；被判为未知的样本进入后续未知类细分模块。

## 8. 当前Oracle结果

正式采用的版本为五折自动选出的：

`s1.5_l1.0_e250_r43_q0.96`

最终指标为：

| 指标 | 结果 |
| --- | ---: |
| Known Acc. | 94.55% |
| Unknown Recall | 96.52% |
| Macro F1 | 92.81% |
| AUROC | 98.14% |

这一结果满足当前实验约束：

- Known Acc.不低于93%；
- Unknown Recall不低于95%；
- Macro F1不低于91%。

## 9. 论文中可以如何表述

可以将该模块概括为：

> 本文首先在已知类嵌入空间中挖掘原型竞争边界样本，并沿类别外侧方向生成特征层伪未知样本。随后，以已知验证样本作为负类、伪未知样本作为正类，监督训练轻量级双重校准器。该校准器联合建模OpenMax未知分数、原型距离未知分数及其线性融合结果，输出最终未知概率。模型超参数和按类别阈值通过五折留类模拟未知协议自动确定，真实未知类仅用于冻结参数后的最终评价。
