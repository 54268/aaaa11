# 开放集 SEI 项目详细说明

这份文档用于说明当前项目的完整方案逻辑、数据流、方法细节和输出含义。根目录的 `README.md` 保持简洁，只负责入口、目录和指标速查；本文件负责解释“这个项目为什么做、做了什么、数据怎么流动、每一步生成什么”。

本文参考了早期方案汇报 PPT，但以当前代码实现为准。和早期 PPT 相比，当前版本已经从“开放集拒识”扩展到“拒识后的未知类细分”，并且项目结构已经整理为根目录入口、`settings/` 默认参数、`functions/` 函数实现三部分。

## 1. 项目要解决的问题

特定发射极识别，也就是 SEI，通常希望根据无线设备发射信号中的射频指纹来判断信号来自哪个具体发射源。传统闭集识别默认测试阶段出现的类别都已经在训练阶段见过，因此模型一定会把输入样本分到某个已知发射源。

真实开放电磁环境并不满足这个假设。测试阶段可能出现训练时没有见过的新发射源、异常发射源或干扰源。如果仍然使用闭集分类器，未知发射源很容易被强行分到某个已知发射源，这会带来认证误判和监测风险。

本项目围绕两个层次的问题展开：

1. 开放集拒识：当输入样本来自未知发射源时，模型应当拒绝把它判成已知类，而是输出 unknown。
2. 未知类细分：对已经被拒识为 unknown 的样本，进一步判断它们内部可能来自哪些不同未知发射源。

因此项目不是单纯追求闭集分类准确率，而是同时关心已知类识别、未知类拒识、以及拒识后未知样本的聚类可分性。

## 2. 当前方案概览

当前方法可以概括为：

基于原型竞争边界挖掘的开放集发射源识别方法，并在 Oracle 与 WiSig 两个数据集上扩展为已知原型引导的半监督聚类未知类细分（支持 KMeans / Agglomerative-Cosine / GMM 多后端切换）。

整体流程如下：

1. 将原始 WiSig 或 Oracle 信号整理为统一的 `.npz` 数据划分。
2. 只用已知类训练闭集特征提取器和原型分类头。
3. 在特征空间中计算每个已知类的原型。
4. 挖掘靠近分类边界、且容易和其他类别发生竞争的边界样本。
5. 从边界样本向类外侧外推，生成伪未知样本。
6. 用验证集已知样本和伪未知样本校准 OpenMax 与原型距离未知分数。
7. 融合 OpenMax 未知分数和原型距离未知分数，得到最终 unknown score。
8. 根据全局阈值或按类别阈值进行开放集拒识。
9. 对 Oracle 和 WiSig 中被拒识为 unknown 的样本构造细分特征，并进行未知类细分；两个数据集当前统一使用 `embedding_stats + PCA96 + unknown-only GMM-full-direct` 口径。细分时将 GMM 候选分量数写成 `K+m`，其中 `K` 为协议给定未知类数，`m∈{0,1,2,3}` 为小范围冗余候选因子。离线敏感性分析采用最小充分冗余规则选择 `m`：最终有效簇数必须等于 K，且覆盖率修正后的细分质量提升超过 1 个百分点才增加结构。当前 Oracle 选择 `m=2`，WiSig 选择 `m=0`。

当前项目中的主要入口为：

- `run_wisig.py`：WiSig 开放集拒识实验，默认串联未知类细分。
- `run_oracle.py`：Oracle 开放集拒识实验，默认串联未知类细分。
- `run_oracle_subdivision.py` / `run_wisig_subdivision.py`：基于已有闭集权重，单独刷新未知类细分。

## 3. 数据进入项目后的格式

项目最终训练和评估都使用统一的四个 `.npz` 文件：

- `train_known.npz`：已知类训练集。
- `val_known.npz`：已知类验证集。
- `test_known.npz`：已知类测试集。
- `test_unknown.npz`：未知类测试集。

每个 `.npz` 至少包含：

- `x`：I/Q 信号片段，形状为 `[样本数, 2, 信号长度]`，其中第 0 个通道是 I 分量，第 1 个通道是 Q 分量。
- `y`：整数标签。已知类为 `0` 到 `K-1`，未知类文件中通常为 `-1`。
- `label_name`：原始类别名称，仅用于报告、混淆分析和离线评估。

训练时只使用 `train_known.npz` 和 `val_known.npz` 中的已知类样本。`test_unknown.npz` 不参与闭集训练，也不参与 OpenMax 拟合，只在最终开放集评估和未知类细分评估时使用。

## 4. WiSig 数据处理

WiSig 当前使用 `data/raw/wisig/SingleDay.pkl` 作为原始输入，处理逻辑在 `functions/data/prep_wisig.py`。

处理过程如下：

1. 读取 WiSig compact pickle 数据。如果原始文件是压缩包，会先自动解包。
2. 按 split 文件指定的发射源列表划分已知类和未知类。
3. 按接收机、采集日期、equalized 标记筛选样本。当前主实验采用受控条件，避免把过多接收机和日期因素混进主实验。
4. 将 WiSig 数据从原始排列转换为 `[N, 2, L]` 的 I/Q 格式。
5. 每个已知类按训练、验证、测试比例分层划分。
6. 未知类样本全部放入 `test_unknown.npz`。
7. 写出数据统计、类别划分和处理清单。

当前 WiSig 主协议大致为：

- 发射源总数：28。
- 已知类：16。
- 未知类：12。
- 固定接收机：`1-1`。
- 训练已知样本：8960。
- 验证已知样本：1280。
- 测试已知样本：2560。
- 测试未知样本：9600。

WiSig 上拒识较容易，当前通过手动全局阈值 `0.94` 将未知类召回率提高到 1.0。早期版本中未对 WiSig 做未知类细分，现已补齐：当前通过 `embedding_stats` 特征 + PCA96 + `gmm_full_direct` 后端 + 12 个目标未知簇完成细分，未知类细分指标仍可保持 NMI ≈ 0.998。

## 5. Oracle 数据处理

Oracle 当前使用 SigMF 文件作为原始输入，处理逻辑在 `functions/data/prep_oracle.py`。

处理过程如下：

1. 扫描 `data/raw/oracle/KRI-16IQImbalances-DemodulatedData` 下的 `.sigmf-meta` 文件。
2. 根据元数据找到对应的 `.sigmf-data` 文件。
3. 根据 SigMF 中声明或推断的数据类型读取复数 I/Q 信号。
4. 从文件名中解析发射源类别。
5. 将连续复数信号按固定长度切成片段，当前信号长度为 256。
6. 每个片段被整理为 `[2, 256]`，分别表示 I/Q 两路。
7. 按 split 文件指定的已知类和未知类划分。
8. 已知类分层切分为训练、验证、测试；未知类全部进入测试未知集。

当前 Oracle 主协议为：

- 发射源总数：16。
- 已知类：10，分别为 `1, 2, 3, 4, 5, 7, 9, 13, 14, 15`。
- 未知类：6，分别为 `17, 18, 19, 25, 26, 32`。
- 训练已知样本：28000。
- 验证已知样本：4000。
- 测试已知样本：8000。
- 测试未知样本：24000。

## 6. 样本归一化和加载

数据加载逻辑在 `functions/data/npz_dataset.py` 和 `functions/data/data_build.py`。

每个样本取出时可以做逐样本归一化。当前主要设置为 `per_sample`，即对每个样本的 I/Q 两个通道分别减均值、除标准差：

- 这样可以降低整体幅值差异对模型的影响。
- 模型更关注波形结构和射频指纹差异。
- 归一化发生在 DataLoader 取样阶段，不改写磁盘上的 `.npz` 文件。

数据模块统一提供四个加载器：

- 训练已知类加载器。
- 验证已知类加载器。
- 测试已知类加载器。
- 测试未知类加载器。

## 7. 闭集模型

闭集模型由两个部分组成：

1. 复值卷积特征提取器。
2. 原型分类头。

### 7.1 复值卷积特征提取器

实现位置为 `functions/model/cvcnn_iq.py`。

输入是 `[B, 2, L]` 的 I/Q 信号。模型先把 I/Q 两个实值通道组合成复数表示，然后经过三层复值卷积块：

- 复值卷积。
- 实部和虚部分别做 BatchNorm。
- 实部和虚部分别做 ReLU。
- 平均池化降采样。
- dropout 抑制过拟合。

最后对实部和虚部做全局平均池化，将二者拼接后经过全连接投影，得到固定维度的 embedding。当前 embedding 维度为 128，并会做 L2 归一化。

### 7.2 原型分类头

实现位置为 `functions/model/prototype_head.py`。

每个已知类维护一个原型向量。分类时计算样本 embedding 到各类原型的平方欧氏距离，距离越近，类别 logit 越大：

- 距离：样本特征到类别原型的平方欧氏距离。
- logit：距离取负后除以温度系数。
- 预测类别：距离最近的原型对应类别。

训练过程中，原型会根据同类样本 embedding 的均值进行更新。每个 epoch 后还会重新用训练集 embedding 刷新原型，使原型更稳定地代表已知类中心。

## 8. 训练目标

闭集训练只使用已知类样本。损失由三部分组成，实现在 `functions/model/closed_set.py`。

1. 交叉熵损失：保证已知类分类能力。
2. 角度间隔损失：鼓励样本和本类原型更接近，同时和最近异类原型拉开差距。
3. 原型紧致损失：使用 `MSE(z_i, p_y)` 鼓励样本 embedding 在欧氏距离空间内靠近本类原型。

当前默认权重为：

- `lambda_basic = 1.0`。
- `lambda_angle = 0.15`。
- `lambda_prototype = 0.1`。
- `angle_margin = 0.15`。

当前消融结果显示，原型紧致损失不是稳定增益项：由于分类头本身已经使用样本到类别原型的平方欧氏距离构造 logits，交叉熵已经隐式约束了原型距离；额外的 `MSE(z_i, p_y)` 在 Oracle 上会与角度间隔目标产生重复约束。因此论文中更适合把角度间隔作为闭集表征学习的主要增强项，原型紧致损失作为诊断性消融结果说明，而不是写成必须启用的核心贡献。

这一步的核心目的不是直接拒识未知类，而是先获得结构更清晰的已知类特征空间。后面的边界挖掘、伪未知生成、OpenMax 和原型距离都依赖这个特征空间。

## 9. 原型竞争边界挖掘

实现位置为 `functions/methods/boundary_mining.py`。

PPT 中提到的“边界样本”和“关键边界样本”现在按公式中的两个阶段筛选：

1. 先用局部边缘性得分 `local_marginality` 得到 MBS。
2. 再在 MBS 内用原型竞争距离 `competition_distance` 得到 PCBS。

### 9.1 局部边缘性

对于样本 `x_i`，代码计算：

```text
s_i^m = ||z_i - p_y||^2 + beta * mean_{j in N_k(i)} ||z_i - z_j||^2
```

其中第一项是原型偏离，第二项是同类 KNN 的局部稀疏性。当前 `k = 5`，`beta = 1.0`，配置位置为 `settings/*_settings.py` 的 `boundary.beta`。

每个类别内部会单独取 `local_marginality` 的阈值 `marginal_threshold`，因此不同类别的 MBS 阈值可以不同。

### 9.2 原型竞争距离

对每个样本，先找到最近异类原型，再计算它到本类原型和最近异类原型等距超平面的有符号距离：

```text
b_i = (||z_i - p_c||^2 - ||z_i - p_y||^2) / (2 ||p_c - p_y||)
```

- `b_i >= 0`：样本仍在本类原型一侧，可以作为边界候选。
- `b_i < 0`：样本已更靠近异类原型，被视为异常或噪声候选，代码会写入 `noise_mask`，并从 MBS/PCBS 及伪未知生成中排除。

### 9.3 PCBS 筛选

代码先按类选出 MBS，然后在每个类别的 MBS 内选择 `competition_distance` 最小的 `top_m` 个样本作为 PCBS。这个过程等价于为每个类别得到一个独立的 `competition_threshold`：

```text
PCBS_c = {i in MBS_c | 0 <= b_i <= competition_threshold_c}
```

`score = alpha * marginality_norm + (1 - alpha) * (1 - competition_norm)` 仍会保留，用于伪未知外推强度的自适应调整，但不再作为 PCBS 的唯一筛选依据。

### 9.4 两类边界样本

当前会筛出两类边界样本：

- 关键边界样本：每个已知类中非负竞争距离最小的 `top_m` 个 MBS，当前 Oracle 和 WiSig 均为 50。
- 普通边缘样本：每个已知类中局部边缘性较高但不属于关键边界的 MBS，比例由 `ordinary_edge_ratio` 控制，当前为 0.15。

这一步输出：

- `boundary_mining.npz`：embedding、原型、`local_marginality`、`competition_distance`、`noise_mask`、各类 mask 等数组。
- `boundary_summary.json`：每个类别筛出的 MBS/PCBS 数量、被排除的负 `b_i` 样本数、按类阈值等信息。

## 10. 伪未知样本生成

实现位置为 `functions/methods/pseudo_unknown.py`。

伪未知样本不是从原始 I/Q 波形生成，而是在训练好的 embedding 空间中生成。它的作用是为 OpenMax 和融合阈值校准提供“未知类边界外侧”的辅助样本。

### 10.1 普通边缘样本外推

对普通边缘样本，生成方向为：

```text
样本 embedding - 本类原型
```

也就是从本类原型指向类外侧。步长由局部尺度、基础系数和边界得分共同决定。边界得分越高，外推略强。

### 10.2 关键边界样本外推

关键边界样本同时考虑两个方向：

- 本类法向方向：从本类原型指向样本。
- 异类排斥方向：从最近异类原型指向样本。

二者按 `critical_beta` 融合。当前 `critical_beta = 0.7`，表示更重视本类边界外侧方向，同时保留异类竞争信息。

### 10.3 冲突保护

如果本类法向方向和异类排斥方向发生明显冲突，代码会对排斥方向做正交化处理，避免外推方向反而指回已知类区域。这对应 PPT 中的“局部方向冲突保护”。

### 10.4 自适应步长和扰动

伪未知生成会使用：

- 局部 KNN 尺度作为基础步长。
- 普通边缘样本和关键边界样本各自的外推系数。
- 边界得分控制的倍数修正。
- 小范围随机 jitter，避免所有伪未知样本落在单一射线上。

这一步输出：

- `pseudo_unknown.npz`：伪未知 embedding、来源样本索引、来源类别、伪未知类型。
- `pseudo_unknown_summary.json`：普通伪未知、关键边界伪未知和总数统计。

## 11. OpenMax 校准

实现位置为 `functions/methods/openmax_wrapper.py`，调用位置在 `functions/pipeline.py`。

OpenMax 的基本思想是：对于每个已知类，统计该类正确识别样本的激活向量或距离分布尾部，当测试样本离已知类分布尾部过远时，提高它属于 unknown 的概率。

当前项目中：

1. 使用验证集已知样本提取 embedding。
2. 计算样本到各类原型的距离。
3. 将距离转换为类似激活向量的表示。
4. 对每个类别拟合 OpenMax 所需的尾部分布。
5. 在测试阶段输出 `q_om`，即 OpenMax unknown probability。

当前 Oracle 和 WiSig 默认使用仓库内 OpenMax 后端 `repo_openmax`。其可追踪源码位于
`functions/methods/repo_openmax/`，不再依赖第三方子模块中的未跟踪 `.pyc` 文件。主要参数为：

- `alpha_rank = 3`。
- `tail_size = 25`。
- `distance_type = eucl`。

## 12. 原型距离未知分数

OpenMax 之外，项目还计算一个原型距离未知分数，实现在 `functions/methods/fusion.py`。

对样本先用最近原型得到闭集预测类别，再取它到该预测类别原型的距离。验证集上会统计每个类别的距离均值 `mu` 和标准差 `sigma`。测试时使用标准化距离：

```text
normalized = (d_min - mu[class]) / sigma[class]
```

再经过 sigmoid 得到 `q_pd`：

```text
q_pd = sigmoid(normalized)
```

直观解释：

- 样本离预测类别原型越近，越像已知类，`q_pd` 越低。
- 样本离预测类别原型越远，越可能是未知类，`q_pd` 越高。

## 13. 未知分数融合和拒识判决

最终 unknown score 由 OpenMax 分数和原型距离分数融合：

```text
q_u = lambda * q_om + (1 - lambda) * q_pd
```

当前 Oracle 最终设置为：

- `lambda = 0.35`。
- 阈值模式：`manual_classwise`。
- 分数校准：`classwise_z`，即用验证集已知样本按预测类别统计 unknown score 的均值和标准差，再把测试分数映射到类内 z-score。
- 每个已知预测类别使用一个单独阈值。
- 已知类救回：对已经被拒为 unknown、但仍然非常接近已知原型且原型间隔较大的样本，按配置规则恢复为闭集预测类别。

判决逻辑为：

1. 先用最近原型得到闭集预测类别。
2. 取该样本的最终未知分数 `q_u`。
3. 用验证集得到的类内 z-score 统计量校准 `q_u`。
4. 如果校准后的 `q_u` 大于该预测类别对应阈值，则先输出 unknown。
5. 对被拒样本检查最近原型距离和第一/第二近原型距离间隔；满足已知类救回规则时恢复为闭集预测类别。
6. 其余样本保持 unknown 或闭集预测类别。

WiSig 默认仍保留网格搜索逻辑，Oracle 当前使用已调好的按类别阈值、类内分数校准和已知类救回规则，目的是在当前机器环境下把 `known_accuracy` 提到 0.96 以上，同时尽量保留 `unknown_recall`、`unknown_precision` 和 `auroc`。

## 14. 开放集评估

开放集评估在 `functions/pipeline.py` 中执行，指标实现在 `functions/common/metrics.py`。

测试时会把已知类测试集和未知类测试集合并：

- 已知类真实标签仍为 `0` 到 `K-1`。
- 未知类真实标签统一记为 `K`，也就是 unknown label。

模型输出也在 `0` 到 `K` 之间。这样可以统一统计已知类分类和未知类拒识表现。

当前会输出：

- `open_set_metrics.json`：完整指标。
- `open_set_predictions.csv`：逐样本真实标签、预测标签、unknown score、OpenMax 分数、原型距离分数、最小原型距离。
- `confusion_matrix.csv`：混淆矩阵。
- `final_report.md`：中文结果报告。
- `figures/`：ROC、PR、混淆矩阵、unknown score 直方图、openness 曲线等。

## 15. 未知类细分

未知类细分实现位置为：

- 流程封装：`functions/subdivision_pipeline.py`。
- 聚类方法：`functions/methods/unknown_subdivision.py`。

Oracle 和 WiSig 入口都默认串联未知类细分。两个数据集都先做开放集拒识，把模型判为 unknown 的样本放入 unknown cache，然后只对 unknown cache 做聚类，进一步判断这些样本可能来自哪些不同未知发射源。

### 15.1 unknown cache

先运行开放集拒识。所有被模型判为 unknown 的样本进入 unknown cache。

unknown cache 里理想情况下应该大多是真实未知类，但也可能混入少量被误拒的已知类样本。因此细分时需要考虑已知类污染。

### 15.2 细分特征

当前 Oracle 和 WiSig 使用同一套特征构造接口，并统一采用 `embedding_stats` 特征模式，即闭集模型提取的 128 维 embedding 加每个样本自身的 I/Q、幅度和相位差统计特征。前期诊断表明，在当前 Oracle 协议下，距离向量会加重 17/25、26/32 等未知类混合，而原始 I/Q 统计量能显著补充未知发射源之间的可分信息。同步到 WiSig 后，细分指标没有明显回退，因此当前两个数据集默认都不再拼接已知原型距离。

也就是说，细分不是直接对原始 I/Q 波形聚类，而是在闭集模型学习到的深度 embedding 空间中加入轻量 I/Q 统计信息后聚类；是否拼接已知原型距离仍可由 `unknown_subdivision.feature_mode` 切换，但当前主结果统一采用 `embedding_stats`。

`functions/subdivision_pipeline.py` 也实现了其他备选特征模式：`embedding`、`score_distance`、`embedding_score_distance`、`prototype_residual`、`residual_distance`，可以通过 `unknown_subdivision.feature_mode` 切换。

### 15.3 标准化和 PCA

细分前会拟合特征预处理器。若当前数据集启用了已知原型锚点，则会将样本特征和已知原型锚点特征合在一起拟合；若关闭已知原型锚点，则只使用 unknown cache 样本特征拟合：

- 先做标准化。
- 再做 PCA 降维。

当前 Oracle 设置：

- `feature_mode = embedding_stats`。
- `pca_dim = 96`。
- `clustering_backend = gmm_full_direct`。
- `k_min = k_max = 6`，`target_num_clusters = 6`（与 Oracle 协议中的 6 个真实未知发射源对齐）。
- `overcluster_extra_candidates = [0, 1, 2, 3]`，在小范围候选中自动选择冗余候选分量数。
- 当前离线选择 `overcluster_extra_clusters = 2`，即 `m=2`，实际先拟合 `K+m=8` 个候选分量。
- `direct_confidence_quantile = 0.10`，GMM 后验置信度最低的 10% 样本标为不确定。
- `direct_min_cluster_size = 800`，候选分量小于 800 个样本时视为不稳定小簇并标为不确定。
- `use_known_prototype_anchors = False`，Oracle 当前主要依赖 GMM 置信度和小簇剔除做细分不确定过滤，不再额外用已知锚点距离过滤。

当前 WiSig 设置：

- `feature_mode = embedding_stats`。
- `pca_dim = 96`。
- `clustering_backend = gmm_full_direct`。
- `k_min = k_max = 12`，`target_num_clusters = 12`（与 WiSig 当前 12 个真实未知发射源对齐）。
- `overcluster_extra_clusters = 0`，即 `m=0`，直接拟合 `K+m=12` 个候选簇；实测更大的冗余因子会把两个干净未知类拆成类内子模态，导致匈牙利准确率回退。
- `direct_confidence_quantile = 0.10`，GMM 后验置信度最低 10% 的样本可标为不确定；当前 WiSig 细分中各样本后验较稳定，最终没有样本被过滤。
- `direct_min_cluster_size = 160`，按 Oracle 小簇阈值约 20% 单类规模的口径折算到 WiSig。
- `use_known_prototype_anchors = False`，与 Oracle 保持相同的 unknown-only 预处理与细分口径。

### 15.3.1 类内子模态和冗余候选分量

类内子模态指的是：同一个真实未知发射源的样本，在特征空间里不一定只形成一个紧凑样本团，而可能因为信号切窗、幅度/相位扰动、I/Q 统计差异、噪声或边界样本等因素分成多个局部小团。真实标签上它们仍属于同一个未知发射源，但从 GMM 的角度看，它们可能像多个局部分布。

例如 Oracle 有 6 个真实未知类。理想情况下，6 个未知类在特征空间中对应 6 个样本团，GMM 拟合 6 个分量就足够。但如果未知类 A 内部被分成 A1 和 A2 两个局部小团，特征空间里实际看起来可能接近 7 个团：`A1, A2, B, C, D, E, F`。如果仍只给 GMM 6 个分量，某一个分量可能被 A2 这种局部子团占用，导致另两个真实未知类被迫合并到同一个分量里。

因此当前方法把候选分量数统一写成 `K+m`：`K` 是协议给定的未知类数量，`m` 是冗余候选因子。`m` 不是新的未知类数量，而是给 GMM 留出的缓冲容量，用于吸收低置信样本、边界样本和不稳定子模态。本文只在一个很小的集合 `m∈{0,1,2,3}` 中进行离线敏感性分析：候选必须最终保留 K 个有效簇，并使用覆盖率修正后的 NMI、ARI、Purity、Hungarian Accuracy 均值衡量增益；新增一个冗余分量带来的增益必须超过 1 个百分点才接受。

当前 Oracle 选择 `m=2`：`m=0` 只解析出 5 个有效簇，不满足 K=6；`m=1` 能恢复 6 个有效簇，但覆盖率修正质量为 `0.805952`；`m=2` 在最终仍为 6 个有效簇的前提下提高到 `0.839077`，增益约 3.31 个百分点，超过 1% 门槛；`m=3` 降至 `0.816642`，因此没有继续增加结构。

WiSig 则相反：当前 SingleDay 受控条件下未知类本身非常干净。同步到 `embedding_stats + PCA96` 后，如果采用更大的冗余因子，GMM 会把 `8-18`、`8-20` 这类本来纯净的未知类拆成两个较大的类内子模态，导致有效簇数超过协议目标，匈牙利准确率从约 0.999 回退到约 0.922。因此 WiSig 正式主结果选择 `m=0`，直接使用协议目标 `K=12`。

### 15.4 已知原型引导的半监督聚类

细分主入口是 `functions/methods/unknown_subdivision.run_ofscil_subdivision`，按指定的 `clustering_backend` 拟合初始中心，再做带原型约束的迭代再分配。当前支持的后端：

| `clustering_backend` | 说明 | 适合什么数据 |
| --- | --- | --- |
| `kmeans` | 球面 KMeans（先 L2 归一化，再用 KMeans++）。 | 各簇大小相近、近似球面分布的样本。 |
| `agglomerative_cosine` | 凝聚式聚类，子采样 + cosine 平均连接得到初始中心。 | 簇形不规则、不同簇方差差距大的候选设置。 |
| `gmm` / `gmm_diag` | 对角协方差高斯混合估计中心，再做原型引导分配。 | 簇是椭球状但方向接近坐标轴。 |
| `gmm_full` | 全协方差高斯混合估计中心，再做原型引导分配。 | 各簇方向不一致、形状差异大。 |
| `gmm_full_direct` | 直接用 GMM-full 预测标签，跳过余弦再分配。 | 各簇并不严格分布在球面上的情形。 |

除 `gmm_full_direct` 外，其他后端会进入原型引导迭代，核心逻辑为：

1. 对 unknown cache 特征拟合初始中心。
2. 每轮计算样本到未知聚类中心的余弦距离。
3. 同时计算样本到已知原型锚点的余弦距离。
4. 如果样本离已知锚点过近（满足 `cos_dist(known) ≤ cos_dist(unknown) + known_reject_margin`），则标记为 suspected known，不强行分入未知簇，而是置为不确定样本 `-1`。
5. 对已分配到未知簇的样本更新聚类中心。
6. 重复直到标签不再变化或达到迭代上限。

`gmm_full_direct` 会直接使用 GMM-full 的预测标签，跳过余弦再分配；如果启用了已知原型锚点，则还会结合已知原型锚点距离做疑似已知污染过滤，把过近的样本标记为不确定 `-1`。Oracle 当前关闭该锚点过滤，避免 `embedding_stats` 中原型锚点缺少 I/Q 统计量而干扰未知细分。

Oracle 和 WiSig 当前都在 `gmm_full_direct` 上保留两步过滤能力：

1. 低置信过滤：根据 GMM 后验概率，把后验置信度较低的样本标为不确定 `-1`。
2. 小簇剔除：如果过聚类产生的候选簇样本数过小，则认为它是不稳定簇或已知污染簇，不作为最终未知细分类保留。

因此报告中会同时出现 `target_num_clusters` 和 `fit_num_clusters`：前者是协议目标未知发射源数量 `K`，后者是实际拟合的候选簇数 `K+m`。当前 Oracle 为 `K=6, m=2, fit_num_clusters=8`，通过两个冗余候选分量吸收不稳定样本；WiSig 为 `K=12, m=0, fit_num_clusters=12`，不额外过聚类。最终指标以 `resolved_num_clusters` 为准，当前 Oracle 保留 6 个有效未知细分类，WiSig 保留 12 个有效未知细分类。

K 的自动搜索仍然保留：当 `target_num_clusters` 没设置或 `target_k_strength < 1.0` 时，会在 `[k_min, k_max]` 范围内联立轮廓系数、Davies-Bouldin、Calinski-Harabasz、跨种子稳定性、不确定率惩罚、中心-已知原型距离惩罚等指标自动挑选 K。本项目入口默认用 `k_min = k_max` 把 K 锁定到经验最优值。

## 20. 一次完整实验的数据流

下面按数据流总结一次 Oracle 拒识加细分实验。

1. 原始 SigMF 文件进入 `prepare_oracle_sigmf`。
2. 复数 I/Q 信号被切成 `[2, 256]` 片段。
3. 按 split 文件生成 `train_known`、`val_known`、`test_known`、`test_unknown`。
4. DataLoader 读取 `.npz`，对每个样本做逐样本归一化。
5. CVCNN 将 I/Q 信号映射成 128 维归一化 embedding。
6. 原型分类头根据 embedding 到原型的距离进行闭集分类。
7. 训练结束后保存 `best_closed_set.pt`。
8. 用训练集 embedding 挖掘边界样本，保存 `boundary_mining.npz`。
9. 用边界样本在 embedding 空间生成伪未知，保存 `pseudo_unknown.npz`。
10. 用验证集已知样本拟合 OpenMax，保存 `openmax.pkl`。
11. 统计每类原型距离分布，保存 `distance_stats.npz`。
12. 用验证集已知样本和伪未知样本校准融合策略，保存 `fusion.json`。
13. 对测试已知和测试未知样本计算 unknown score，输出开放集指标和报告。
14. 将被判为 unknown 的样本组成 unknown cache。
15. 为 unknown cache 构造细分特征；Oracle 与 WiSig 当前都使用 `embedding_stats`。
16. 标准化、PCA 降维。
17. 用 `gmm_full_direct` 后端完成细分；Oracle 会在 `m=0,1,2,3` 中按离线最小充分冗余规则选择 `m=2`，先拟合 8 个候选分量，再通过低置信过滤和小簇剔除保留 6 个有效未知细分类；WiSig 选择 `m=0`，直接拟合 12 个候选簇并保留 12 个有效未知细分类。
18. 输出未知类细分指标、分配表和混淆表。

WiSig 的端到端流程与 Oracle 类似，也同步使用 `embedding_stats + PCA96 + unknown-only GMM-full-direct`；区别在于 WiSig 数据更容易，额外过聚类会产生超过 K 的最终有效簇，因此最小充分冗余规则选择 `m=0`。当前同步后仍能保持接近满分的细分指标。

## 21. 后续优化方向

当前项目已经能完整完成数据处理、闭集训练、开放集拒识、未知类细分和报告输出。后续如果继续优化，可以主要从以下方向做：

1. 训练侧：调整闭集 embedding 的类间间隔和类内紧致性，让 Oracle 已知类准确率更稳。
2. 拒识侧：继续搜索融合分数、类内校准、按类别阈值和已知类救回规则，使 `known_accuracy`、`unknown_recall`、`unknown_precision`、`macro_f1`、`fpr95` 的权衡更好。
3. 伪未知侧：调整关键边界样本数量、普通边缘比例、外推步长和冲突保护强度。
4. 细分侧：继续比较 `score_distance`、`embedding_distance`、`residual_distance`、过聚类候选簇数、置信度过滤分位数和小簇阈值等设置。
5. 对比实验：将 OpenMax、ACGAN+OpenMax、OFSCIL、EPD/IOWL 等方法整理为同协议下的 baseline。
