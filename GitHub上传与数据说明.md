# GitHub 上传与数据说明

## 当前仓库上传策略

- 本次上传到 GitHub 的内容只包含代码、配置、说明文档和运行入口。
- `data/` 整个目录不会上传到 GitHub。
- `outputs/`、`figures/`、`__pycache__/` 和 `third_party/` 下的大体积克隆目录也不会上传。

## 为什么要忽略 `data/`

- 你的 `data/` 目录包含真实数据集，体积较大。
- 数据集通常不适合直接放进代码仓库，尤其是私有实验工程刚整理阶段。
- 这样做可以让仓库更轻，推送更稳，也避免误把十几个 GB 的数据推上去。

## 会不会影响本地运行

- 不会。
- 忽略 `data/` 只影响 Git 是否跟踪这些文件，不会删除你本地磁盘里的数据。
- 你本地现有的训练、评测、绘图流程仍然会继续读取 `E:\learn_pytorch\笔记\方案\os_sei_code\data\` 下的数据。

## 以后在别的机器上怎么恢复

- 先把本仓库代码拉下来。
- 再手动把 WiSig / Oracle 数据放到 `os_sei_code/data/` 下。
- 如果需要，也可以重新运行项目里的数据划分脚本与预处理脚本来生成实验划分。

## 说明

- 这次 GitHub 上传不会改动你的原始数据内容。
- 这次 GitHub 上传不会影响你本机已有的模型训练与测试结果。
