# 图表与结果索引

## 1. 先看哪里

如果你现在只想快速判断结果，建议按这个顺序看：

1. `RESULT_SUMMARY.md`
2. `figures/`
3. 对应实验目录下的 `final_report.md`
4. 需要更细时再看 `open_set_metrics.json / open_set_predictions.csv`

## 2. 当前推荐主结果

### Oracle 已知类优先版

- 配置：`configs/oracle_sigmf_known_first.yaml`
- 输出：`outputs/oracle_kri16_demod_known_first/`
- 图表：`figures/oracle_kri16_demod_known_first/`

核心指标：

| 指标 | 数值 |
| --- | ---: |
| 总体准确率 | 0.613125 |
| 已知类准确率 | 0.953250 |
| Macro-F1 | 0.769000 |
| AUROC | 0.954129 |
| FPR95 | 0.154750 |
| 未知类召回率 | 0.499750 |

解读：

- 这版是当前最符合“已知类优先”的 Oracle 主结果。
- 已知类准确率已经超过 95%。
- 未知类召回率不再追求极端高值，而是为了保住已知类精度做了平衡。

### WiSig 四子集平衡版

- 配置：`configs/wisig_all_compact_balanced.yaml`
- 输出：`outputs/wisig_all_compact_balanced/`
- 图表：`figures/wisig_all_compact_balanced/`

核心指标：

| 指标 | 数值 |
| --- | ---: |
| 总体准确率 | 0.448090 |
| 已知类准确率 | 0.626042 |
| Macro-F1 | 0.472241 |
| AUROC | 0.679912 |
| FPR95 | 0.967083 |
| 未知类召回率 | 0.412500 |

解读：

- WiSig 四子集联训联测已经真实接入并跑通。
- 但这条线目前仍是“挑战性结果”，不能作为最终主结论。
- 当前主要瓶颈不是开放集阈值，而是跨子集条件下闭集表征还不够强。

## 3. 图表目录里有什么

每个实验的图表目录下当前会自动生成：

1. `roc_curve.png`
   - 开放集 ROC 曲线

2. `pr_curve.png`
   - 开放集 PR 曲线

3. `confusion_matrix.png`
   - 混淆矩阵图

4. `unknown_score_histogram.png`
   - 已知类与未知类未知分数分布图

5. `openness_metric_curve.png`
   - Openness–Metric 折线图

6. `openness_curve.json`
   - Openness 曲线原始数值

## 4. 最常用入口

### 跑 Oracle 主结果

```bash
python run_oracle_demo.py
```

### 跑 WiSig 主结果

```bash
python run_wisig_demo.py
```

说明：

- `run_oracle_demo.py` 现在默认指向 `oracle_sigmf_known_first.yaml`
- `run_wisig_demo.py` 在检测到四个 WiSig 子集都存在时，默认指向 `wisig_all_compact_balanced.yaml`

## 5. 你现在该怎么看

如果你是为了判断“方案是否符合目标”，建议这样看：

1. 先看 Oracle 主结果
   - 判断已知类优先目标是否达成

2. 再看 Oracle 图表
   - 尤其看 ROC、分数分布图、Openness 曲线

3. 再看 WiSig 四子集结果
   - 把它当作跨数据集验证
   - 目前更像发现问题和后续继续优化的依据

4. 最后再看 `模型演进与仓库适配说明.md`
   - 核对整套工程是怎么从 GitHub 仓库适配到你方案的
