# 对比方法核心流程简述

**Softmax。** 使用紧凑型 1D I/Q CNN 提取归一化特征并训练已知类 Softmax 分类器；测试时以验证集已知样本的最大 Softmax 概率分位数确定拒识阈值，低于阈值的样本判为未知类。

**OpenMax。** 先训练与 Softmax 相同的闭集分类器，再用正确分类训练样本的激活向量拟合各类均值激活向量和 Weibull 尾部分布；测试时重标定已知类概率并分配未知类概率，最后依据验证集确定的阈值完成拒识。

**HyperRSI。** 当前正式适配采用紧凑复数卷积网络提取单位超球面特征，并用角度间隔分类头训练已知类；测试时计算样本与各已知类中心的余弦相似度，以验证集分位数阈值拒识未知样本。代码保留了 GPD 尾部建模实现，但当前正式入口使用类中心余弦评分。

**HyDRA。** 采用时域/频域并行残差卷积提取局部特征，再由 Transformer 和分类头完成已知类学习；开放集阶段按数据集使用 Softmax 置信度或类中心相似度，并以验证集阈值判定未知类。

**OpenRFI。** 使用射频扰动与帧重排增强、残差卷积加 Transformer 编码器、ArcMargin 分类头以及可学习原型/分组正则联合训练；拒识阶段进行概率校准，未知类细分阶段在 Oracle 上执行全测试世界的 K+M 原型图聚类和置信度筛选，在 WiSig 上对拒识缓存执行原型分组。

**ARPL。** 通过特征编码器和每类 reciprocal point 构造距离型分类分数，并以分类损失和半径间隔约束训练已知类；正式结果使用 ARPL 输出配合 OpenMax/EVT 风格校准完成拒识。代码也保留 confusing-sample 分支，但默认关闭，因为其在当前 1D I/Q 协议上会损伤已知类准确率。

**NS-RFF。** 原方法先进行同步与频偏补偿，再学习射频指纹特征并以成对距离统计 ROC、AUC 和 EER，属于 ZigBee 认证/验证任务；它与当前“多已知类分类、统一未知类拒识和未知类细分”的评价口径不一致，因此保留原仓库供参考，不纳入主对比表。
