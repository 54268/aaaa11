# 未知类细分流程说明

这份文件专门讲开放集拒识之后的未知类细分，以及标准化和 PCA 到底在做什么。`项目详细说明.md` 只保留主线，细节以后统一看这里。

## 1. 这一步到底放在哪里

当前细分流程发生在开放集拒识之后：

```text
测试样本
  -> 开放集拒识
      -> 判为已知：直接输出已知类
      -> 判为 unknown：进入 unknown cache
  -> 只对 unknown cache 做细分
```

所以细分不是重新给所有测试样本分类，也不是训练阶段的一部分。真实未知标签只在最后离线评估时使用，不参与拒识、特征构造、标准化、PCA 或 GMM 拟合。

## 2. 为什么不是只用 128 维 embedding

这是你问得最关键的一点。

结论先说：**128 维 embedding 由原始 I/Q 序列学习得到，可能包含与幅度变化、相位变化等因素相关的判别信息，但代码没有把这些信息拆成可直接解释的“幅度维”或“相位维”。**
`embedding_stats` 的思路不是重复一遍特征，而是把“学出来的表示”和“手工统计描述”拼在一起，让细分器看到两种互补视角。

### 2.1 128 维 embedding 是什么

它是闭集特征提取器学出来的压缩表示，目标是帮助分类器区分已知发射源。
所谓“包含幅度、相位等信息”，是指改变输入信号的幅度包络、相位演化或频率偏移时，网络输出的 embedding 也可能随之变化。它只能说明 embedding 对这些物理变化有响应，不能说明某一维就等于幅度或相位，也不能从 embedding 中直接还原这些物理量。

### 2.2 为什么还要加统计特征

因为细分阶段关心的不只是“这个样本像不像某个已知类”，还关心：

- 它自己的波形统计长什么样；
- 它的幅度分布是否异常；
- 它的相位变化是否稳定；
- 它和其他 unknown 样本是否共享某种局部形态。

这些变化对 embedding 的影响可能被保留，也可能在面向已知类分类的压缩过程中被弱化。
所以 `embedding_stats` 实际上是在做“深度表示 + 显式统计补充”。

### 2.3 当前实现里到底拼了什么

代码里 `embedding_stats` 不是一句空话，它是：

- 128 维 embedding
- 再加 49 维 I/Q 统计描述

这 49 维来自四组序列统计：

- I 分量统计
- Q 分量统计
- 幅度统计
- 相位增量统计

令复信号为

\[
x_t = I_t + \mathrm{j}Q_t .
\]

代码显式计算的幅度和相位增量分别为

\[
A_t = |x_t| = \sqrt{I_t^2 + Q_t^2},
\qquad
\Delta\phi_t = \arg\left(x_t x_{t-1}^{*}\right).
\]

这里使用相邻采样点的相位增量，而不是容易发生 \(2\pi\) 跳变的绝对相位。对 \(I_t\)、\(Q_t\)、\(A_t\) 和 \(\Delta\phi_t\) 四组序列，代码分别提取均值、标准差、最小值、最大值、偏度、超额峰度以及 5%、25%、50%、75%、95% 分位数，共 \(4\times11=44\) 维；再加入 5 个 I/Q 二阶相关量，最终得到 49 维显式统计描述。

所以当前细分输入总维度是准确的 \(128+49=177\) 维，而不是只看 128 维 embedding。

## 3. 完整流程

```text
原始 I/Q 信号
  -> CVCNN
  -> 128 维 embedding
  -> 开放集拒识
  -> unknown cache
  -> embedding_stats
  -> 标准化
  -> PCA 96 维
  -> GMM-full-direct
  -> 低置信样本和小簇过滤
  -> 有效未知簇或不确定样本 -1
```

当前 Oracle 和 WiSig 都用这套主线。

## 4. 标准化在干什么

标准化的作用很朴素：

- 把每个维度拉到相近尺度；
- 避免某些统计量因为数值大就主导距离；
- 让 embedding 和手工统计特征能够在同一个尺度上竞争。

如果不标准化，PCA 和 GMM 会更偏向数值范围大的维度，结果不稳定。

你可以把它理解成：

> 标准化不是增加信息，而是先把信息放到同一张桌子上。

## 5. PCA 在干什么

PCA 是无监督线性投影。它做的事情是：

- 找出 unknown cache 里变化最大的方向；
- 把冗余相关的维度压缩掉；
- 降低 GMM 估计完整协方差时的难度；
- 让后面的聚类更稳。

## 6. GMM-full-direct

PCA 后的特征送入全协方差 GMM。

这里的意思是：

- full：每个簇都有自己的完整协方差；
- direct：直接用 GMM 的预测标签，不再做余弦距离二次重分配。

为什么不用只看中心距离的 KMeans？

因为 unknown cache 里的簇不一定是圆球状的，可能有拉长、倾斜、相关性强等情况。GMM-full 更能表达这种形状。

## 7. 细分方法对比

为了说明未知类细分模块的必要性，当前细分对比整理为 K-means、HDBSCAN、OpenRFI 和 PCBM (ours) 四类方法。

- K-means：需要给定 K，主要按到中心的距离划分，适合近似球形簇。
- HDBSCAN：层次密度聚类，不依赖固定 `eps`，低密度样本会被标为噪声或不确定样本。
- OpenRFI：不是 DBSCAN/HDBSCAN 这类密度聚类，而是先用 KMeans 形成过完备原型，再在原型相似图上做谱聚类，最后把样本回填到最近原型所在的簇。
- PCBM (ours)：本文完整细分流程，使用 `embedding_stats`、标准化、PCA96、GMM-full-direct、K+m 冗余候选和不确定样本过滤。

Oracle 上，K-means 的匈牙利准确率为 0.921844，HDBSCAN 只解析出 3 个有效簇，OpenRFI 达到 0.930196；PCBM 达到 0.999718，但覆盖率为 0.886292。WiSig 上各方法已接近饱和，PCBM 保持 1.000000 覆盖率。

完整结果见 `outputs/subdivision_backend_comparison/subdivision_backend_comparison.md`。

## 8. K+m 是什么

协议规定真实未知类数是固定的：

- Oracle: 6
- WiSig: 12

但实际特征空间里，一个真实未知类可能被切成几个局部小团，所以我们允许多拟合一点候选分量：

- `K` 是协议规定的未知类数
- `m` 是冗余候选容量
- 实际拟合分量数是 `K + m`

这个 `m` 不是新发现了更多未知类，而是给边界样本、低置信样本和类内子模态留一点缓冲。

当前主结果：

| 数据集 | K | m | 拟合候选数 | 最终有效簇数 |
|---|---:|---:|---:|---:|
| Oracle | 6 | 2 | 8 | 6 |
| WiSig | 12 | 0 | 12 | 12 |

选择规则也很简单：

- 最终有效簇数必须回到 K
- 覆盖率修正后的质量提升要超过 1 个百分点，才值得增加 m

## 9. 一个更短的版本

如果你只想记一句：

> `embedding_stats` 是深度表示加上显式统计特征；标准化负责统一尺度，PCA 负责去冗余和降噪；PCBM 最后通过 GMM-full-direct、K+m 冗余候选和不确定样本过滤完成未知结构划分。
