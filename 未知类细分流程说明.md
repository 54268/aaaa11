# 未知类细分流程说明

本文档说明开放集拒识之后的未知类细分流程。自动候选 K 搜索的专门说明见 `自动K搜索与未知类细分流程说明.md`。

## 1. 流程位置

未知类细分发生在开放集拒识之后，只处理已经被判为 unknown 的样本：

```text
测试样本
  -> 开放集拒识
      -> 已知类：输出已知类别
      -> unknown：进入 unknown cache
  -> 只对 unknown cache 做未知类细分
```

真实未知标签只在最终离线评估时使用，不参与拒识、特征构造、标准化、PCA、GMM 拟合、K 搜索、过滤或合并。

## 2. 特征构造

当前 PCBM 细分输入采用 `embedding_stats` / `embedding_iq_stats` 口径，即深度 embedding 加显式 I/Q 统计特征。

闭集 CVCNN 输出 128 维 embedding。I/Q 统计特征为 49 维，来自 I 分量、Q 分量、幅度、相位增量四组序列统计，以及 5 个 I/Q 二阶相关量。相位增量使用：

```text
delta_phi_t = arg(x_t * conj(x_{t-1}))
```

因此细分特征不是只看 128 维 embedding，而是把深度表征和可解释统计描述拼接后再送入后续步骤。

## 3. 标准化与 PCA

unknown cache 特征先做标准化，再做 PCA96：

```text
embedding + I/Q stats
  -> StandardScaler
  -> PCA 96
```

标准化用于统一不同特征维度的数值尺度，避免某些统计量因量纲较大而主导距离或协方差估计。PCA 是无监督线性投影，用于去冗余、降噪并降低 full-covariance GMM 的估计难度。

## 4. GMM-full-direct

PCA 后的特征送入 full-covariance GMM：

```text
PCA features
  -> GMM-full-direct
  -> candidate labels + posterior confidence
```

`full` 表示每个分量使用完整协方差矩阵；`direct` 表示直接使用 GMM 的预测标签，不再按中心距离做二次重分配。相比 K-means，GMM-full 可以表达拉长、倾斜、相关性较强的簇形状。

## 5. 自动候选 K 搜索

当前不把真实未知类数作为已知 K 提供给模型。细分阶段自动搜索的是 GMM 候选分量数 `fit_K`，最终有效簇数由过滤和合并后得到。

候选 `fit_K` 的无标签选择分数为：

```text
score(fit_K)
  = normalized(GMM lower_bound)
  + 3.0 * cluster_balance
  - 0.03 * fit_K
```

其中 `cluster_balance` 使用过滤/合并后的簇规模均衡度，例如 `mean_cluster_size / max_cluster_size`。这个分数同时避免欠分裂、过分裂和极端不均衡分裂。

## 6. 不确定样本过滤与自动合并

GMM 输出候选标签后，PCBM 会做两类无标签后处理：

1. 低置信/小簇过滤：根据 GMM posterior confidence 和最小簇规模阈值，将不稳定样本标为 `-1`。
2. 簇均衡自动合并：若 `min_cluster_size / mean_cluster_size < 0.75`，将最小簇合并到最近中心，重复直到簇规模恢复均衡。

Oracle 当前自动候选搜索选择 `fit_K=7`，低置信/小簇过滤后得到 `effective_K=6`。WiSig 当前自动候选搜索选择 `fit_K=13`，直接保留 13 簇会把一个真实未知类拆开；簇均衡自动合并 1 次后得到 `effective_K=12`。

## 7. 覆盖率定义

细分表中的 coverage 不是单纯拒识召回率，而是全部真实未知测试样本中最终被分配到有效未知簇的比例：

```text
coverage_of_total_test_unknown
= unknown_cache_recall * coverage_of_selected_true_unknown
```

当细分阶段产生噪声、不确定样本、低置信过滤或置信度筛选时，coverage 会低于 unknown recall。

不同方法的未覆盖来源不同：

- K-means 不输出噪声标签，coverage 通常等于进入评估集合的真实未知样本比例。
- HDBSCAN 在细分聚类阶段把低密度样本标为噪声。
- OpenRFI 采用 full-test-world prototype grouping，并在分组后按置信度保留高置信结果。
- PCBM 的未覆盖样本来自 GMM 后的低置信过滤、小簇过滤和有效簇判定。

## 8. 当前主结果

| 数据集 | NMI | ARI | Hungarian Acc. | Coverage | fit_K | effective_K |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| Oracle | 0.991948 | 0.993851 | 0.997400 | 0.945583 | 7 | 6 |
| WiSig | 0.998125 | 0.998637 | 0.999375 | 1.000000 | 13 | 12 |

旧的 sample-only auto-K 只在抽样 cache 上选 K，会导致 Oracle 欠分裂到 `K=3`，WiSig 欠分裂到 `K=10`。当前结果采用完整候选评估与自动合并后的口径。

## 9. 对比方法口径

细分对比方法包括 K-means、HDBSCAN、OpenRFI 和 PCBM：

- K-means：基于质心距离的划分式聚类；在对比表中仍需指定评估协议使用的候选簇数，因此它不是完全开放场景下的自动发现方法。
- HDBSCAN：密度聚类，不需要固定 K，会输出噪声标签。
- OpenRFI：先形成过完备原型，再在原型相似图上做分组，并按置信度保留结果。
- PCBM：使用 unknown cache、融合特征、标准化、PCA96、GMM-full-direct、自动候选 K 搜索、不确定样本过滤和簇均衡自动合并。

论文中描述 PCBM 时，应避免写成“预先给定未知类数 K”。更准确的表述是：PCBM 自动选择候选分量数 `fit_K`，再由无标签过滤与合并得到最终有效未知簇。
